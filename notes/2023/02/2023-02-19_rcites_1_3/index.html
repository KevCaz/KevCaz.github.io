This was mainly a maintenance release following up on a recent issue in vcr (see Overview of fixes for upcoming v1.2) but I also took the opportunity to improve the set_token() function. Now the user has up to five attempts to set up the token when set_token() is called, after that error is triggered (previously the error was triggered after the first failed attempt).
